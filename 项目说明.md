# 808ServerTo2013 项目说明

## 📋 项目概述

本项目是基于 **JT/T 808-2013** 协议标准的车载终端通讯服务器, 派生自姊妹项目 [`JT808Server2019`](../JT808Server2019/), 主要差异:

- **只解析 2013 版本协议** (字段长度按 2013 规范)
- **检测到 2019 版本数据时, 仅记录设备信息** (手机号 / 协议版本 / 时间 / 消息数 / 远端 IP) 到一个 INI 文件, 不解析消息体, 也不发送 2019 格式应答
- **位置 XML 输出格式与 2019 服务器完全一致**, 上下游对接无差异

---

**派生时间**: 2026-04-08
**基于版本**: JT808Server2019 commit 08969d8 (经纬度上游约定对齐之后)
**协议版本**: JT/T 808-2013 (本服务器); 2019 数据仅记录
**开发语言**: C# (.NET 9.0)
**服务器端口**: 8808 (默认, 区别 2019 服务器的 8809)

## 🎯 核心特性

### 1. 2013 协议解析
- 标准 JT/T 808-2013 字段长度
- 6 字节 BCD 手机号 (10 个数字)
- 5 字节制造商 ID / 20 字节型号 / 7 字节终端 ID
- 鉴权码就是消息体本身 (无长度前缀, 无 IMEI, 无软件版本号)

### 2. 2019 设备记录 (新增)
- 通过消息体属性 bit14 检测是否 2019 版本
- 是 → 路由到 `Version2019Recorder`
- 记录字段:
  - PhoneNumber (BCD 解码后的手机号)
  - ProtocolVersion (2019 协议版本号字段)
  - FirstSeen / LastSeen
  - MessageCount
  - LastMessageId
  - LastRemoteEndPoint
- 持久化文件 `Version2019Devices.ini` (INI 格式, 每台设备一个 [phone] section, 启动时自动加载历史)
- 异步刷盘 (节流 1s/次), 优雅停机最后再 flush

### 3. 完整保留 JT808Server2019 的高并发优化
- SAEA 池化 + ArrayPool 缓冲区
- per-session 发送串行锁
- 异步通道写位置 XML
- 原子时间戳, lock-free 会话管理
- PeriodicTimer 60s 清理超时会话
- 接收循环上限 16 次让出 IOCP 线程

### 4. 完整保留实时位置 XML 存储 + 归档双写
- 字段格式与 2019 服务器**字节级一致** (40 字段, lon/lat 反着标的上游约定也保留)
- 主存储 `LocationData/{phone12位}.xml`, 覆盖写
- 可选归档 `LocationArchive/{yyyyMMdd}/{phone12位}.xml`

## 🚀 快速开始

### 1. 编译

```bash
cd /home/shenzheng/XDW/808ServerTo2013
export PATH="$HOME/.dotnet:$PATH"
./build.sh
# 或: dotnet build --configuration Release
```

### 2. 运行服务器

```bash
cd JT808.Server
dotnet run
```

监听端口 **8808**, 自动加载历史 2019 设备记录。

### 3. 与 2019 服务器并存

两个服务器可以同台机器同时运行 (8808 / 8809), 完全独立。

```bash
# 终端 A
cd /home/shenzheng/XDW/808ServerTo2013/JT808.Server && dotnet run

# 终端 B
cd /home/shenzheng/XDW/JT808Server2019/JT808.Server && dotnet run
```

注意: 同台机器同时运行时, 建议为各自配置不同的 `LocationDataDirectory` 避免文件冲突 (例如 `LocationData2013/` 和 `LocationData2019/`)。

## 📦 2019 设备记录文件示例

```ini
; ============================================================
; 2019 版本设备记录
; 本服务器只解析 JT/T 808-2013 协议
; 检测到 2019 版本数据时, 仅记录设备元信息(不解析消息体)
; 2026-04-08 18:36:30  共 2 台设备
; ============================================================

[013800138000]
ProtocolVersion=0x01
FirstSeen=2026-04-08 18:30:12
LastSeen=2026-04-08 18:35:47
MessageCount=125
LastMessageId=0x0200
LastRemoteEndPoint=172.16.1.10:54321

[014818411623]
ProtocolVersion=0x01
FirstSeen=2026-04-08 18:31:05
LastSeen=2026-04-08 18:36:02
MessageCount=98
LastMessageId=0x0102
LastRemoteEndPoint=172.16.1.20:54322
```

含义:
- `PhoneNumber`: 终端手机号
- `ProtocolVersion`: 2019 协议版本号字段值
- `FirstSeen` / `LastSeen`: 首次/最后见到时间
- `MessageCount`: 累计 2019 消息数
- `LastMessageId`: 最后一次消息 ID
- `LastRemoteEndPoint`: 最后一次远端 IP:Port

运维可以根据这个文件:
- 定位需要降级到 2013 版本的终端
- 跟踪 2019 终端的实际接入情况
- 决定是否需要升级 2013 服务器到 2019 版
